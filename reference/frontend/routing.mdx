# Route reference

All routes are defined in `src/App.jsx` and managed by React Router v6. Protected routes use the `ProtectedRoute` component which checks both authentication status and user role.

## Public routes

These routes are accessible without logging in.

| Route | Component | Description |
|-------|-----------|-------------|
| `/` | `Home` | Landing page with hero, carousel, and featured products |
| `/catalogo` | `Catalog` | Product catalog with filters and sorting |
| `/producto/:id` | `ProductDetail` | Individual product page |
| `/carrito` | `Cart` | Shopping cart |
| `/nosotros` | `About` | About the store |
| `/contacto` | `Contact` | Contact form |
| `/login` | `Login` | Login page |
| `/registro` | `Register` | Registration page |
| `/reportar-fallo` | `BugReport` | Public bug report form |
| `/loader-demo` | `LoaderDemo` | TruckLoader animation demo |

## Protected routes (any authenticated user)

These routes require the user to be logged in but have no role restriction.

| Route | Component | Description |
|-------|-----------|-------------|
| `/checkout` | `Checkout` | Checkout form and order summary |
| `/confirmacion` | `OrderConfirmation` | Post-order confirmation page |
| `/pago` | `Payment` | Payment processing page |

## ProtectedRoute behavior

When a user is not authenticated, `ProtectedRoute` redirects to `/login`, preserving the original destination in `location.state.from` so the user is redirected back after login.

When a user is authenticated but has the wrong role, they are redirected to `/`.

```jsx src/components/ProtectedRoute.jsx
<ProtectedRoute>                         // any logged-in user
<ProtectedRoute roles="client">          // client only
<ProtectedRoute roles={['seller', 'admin']}> // seller or admin
<ProtectedRoute roles="admin">           // admin only
```
