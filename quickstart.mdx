---
title: "Quick start"
description: "Get D&D Textil running locally in minutes"
---

<Tip>
  **Windows users:** A `iniciar_tienda.bat` script is included in the repository root. Double-click it to open the browser and start the frontend dev server in one step — no terminal required.
</Tip>

<Steps>
  <Step title="Prerequisites">
    Make sure you have the following installed before you begin:

    | Requirement | Minimum version |
    |-------------|----------------|
    | Node.js | 18+ |
    | Java | 17+ |
    | PostgreSQL | any recent stable release |
    | Git | any recent stable release |

    Verify your versions:

    ```bash
    node --version    # should print v18.x or higher
    java --version    # should print 17 or higher
    psql --version    # should print psql (PostgreSQL) ...
    git --version
    ```
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Jhonmoreno000/PROYECTO-SENA-TIENDA-TEXTIL-
    cd PROYECTO-SENA-TIENDA-TEXTIL-
    ```
  </Step>

  <Step title="Install frontend dependencies">
    From the repository root, install the Node.js dependencies:

    ```bash
    npm install
    ```

    This installs React 18, Vite, Tailwind CSS, React Router v6, React Hook Form, Zod, Framer Motion, Recharts, and the other packages listed in `package.json`.
  </Step>

  <Step title="Configure the environment">
    The frontend communicates with the Java backend over HTTP. The backend must be reachable at `http://localhost:8081`. This address is hardcoded in `src/context/AuthContext.jsx` (login and register endpoints) and `src/context/ProductContext.jsx` (product CRUD), among others.

    Create a PostgreSQL database for the application and configure your backend's `application.properties` (or `application.yml`) with the database connection details before starting the backend.

    The Vite dev server also proxies `/uploads` requests to `http://localhost:8081`:

    ```javascript vite.config.js
    proxy: {
      '/uploads': {
        target: 'http://localhost:8081',
        changeOrigin: true
      }
    }
    ```
  </Step>

  <Step title="Start the backend">
    Navigate to the backend directory and start the Spring Boot application. The exact command depends on your build tool:

    <CodeGroup>

    ```bash Maven
    cd backend-java
    ./mvnw spring-boot:run
    ```

    ```bash Gradle
    cd backend-java
    ./gradlew bootRun
    ```

    </CodeGroup>

    The API will be available at `http://localhost:8081`. Wait until you see the Spring Boot startup banner in the console before proceeding.
  </Step>

  <Step title="Start the frontend dev server">
    From the repository root, run:

    ```bash
    npm run dev
    ```

    Vite starts the development server and opens the browser automatically. The application is available at:

    ```
    http://localhost:3001
    ```

    <Info>
      The Vite config sets `port: 3001` with `strictPort: false`, so if 3001 is already in use, Vite will pick the next available port and display it in the terminal.
    </Info>
  </Step>

  <Step title="Log in with demo credentials">
    Use the following accounts to explore each role in the application:

    | Role | Email | Password |
    |------|-------|----------|
    | Admin | `admin@ddtextil.com` | `admin123` |
    | Seller | `vendedor@ddtextil.com` | `vendedor123` |
    | Client | `cliente@ddtextil.com` | `cliente123` |

    After a successful login, the app normalizes backend role values (`administrador` → `admin`, `vendedor` → `seller`, `cliente` → `client`) and stores the session in `localStorage` under the key `authUser`.

    <Note>
      Checkout and payment are simulated. No real payment processing takes place, and orders are not charged. Order state is stored in the backend database but no payment gateway is integrated.
    </Note>
  </Step>
</Steps>
