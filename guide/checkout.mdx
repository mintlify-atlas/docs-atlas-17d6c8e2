The checkout page is available at `/checkout`. It is guarded by a `useEffect` that redirects to `/carrito` if the cart is empty, so you can only reach it with at least one item.

<Warning>
  Payment processing is simulated â€” no real financial transactions occur. The payment step collects and stores order data in `localStorage` under the key `lastOrder`, then navigates to `/pago`. No charges are made to any payment method.
</Warning>

## Checkout flow

<Steps>
  <Step title="Review your cart">
    Before proceeding, confirm your items and quantities on the `/carrito` page. Click **Proceder al Pago** in the `CartSummary` to navigate to `/checkout`.
  </Step>
  <Step title="Fill in contact information">
    Enter your full name, email address, and 10-digit phone number. All three fields are required.
  </Step>
  <Step title="Enter your shipping address">
    Provide your street address, city, department (state), and postal code. An optional delivery notes field is available for special instructions.
  </Step>
  <Step title="Confirm the order">
    Click **Confirmar Pedido**. The form validates all required fields. If validation passes, the order data is written to `localStorage` and you are redirected to `/pago`.
  </Step>
  <Step title="Payment page">
    The `/pago` route handles the simulated payment step. On success you are redirected to `/confirmacion` where the order details are displayed and the cart is cleared.
  </Step>
</Steps>

## Checkout form fields

The `CheckoutForm` component (`src/components/CheckoutForm.jsx`) manages a single flat `formData` object initialized in `Checkout.jsx`:

```javascript
const [formData, setFormData] = useState({
  fullName: '',
  email: '',
  phone: '',
  address: '',
  city: '',
  state: '',
  zipCode: '',
  notes: '',   // optional
});
```

### Contact information section

| Field | Type | Required | Validation |
|---|---|---|---|
| `fullName` | text | Yes | Non-empty, minimum 3 characters |
| `email` | email | Yes | Must match `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` |
| `phone` | tel | Yes | Exactly 10 digits (spaces stripped before check) |

### Shipping address section

| Field | Type | Required | Validation |
|---|---|---|---|
| `address` | text | Yes | Non-empty |
| `city` | text | Yes | Non-empty |
| `state` | text | Yes | Non-empty (department name, e.g. *Cundinamarca*) |
| `zipCode` | text | Yes | Non-empty |
| `notes` | textarea | No | Free text, no validation |

## Form validation

Validation runs in two modes:

1. **Real-time** â€” `validateField` is called on every `onChange` event and the error for that field is updated immediately.
2. **On submit** â€” all required fields (everything except `notes`) are validated together before `onSubmit` is called.

```javascript
const handleSubmit = (e) => {
  e.preventDefault();

  const newErrors = {};
  Object.keys(formData).forEach((key) => {
    if (key !== 'notes') { // notes is optional
      const error = validateField(key, formData[key]);
      if (error) newErrors[key] = error;
    }
  });

  setErrors(newErrors);

  if (Object.keys(newErrors).length === 0) {
    onSubmit(); // Proceeds to order submission
  }
};
```

Error messages are displayed in red below each invalid field.

## Order calculation

When you click **Confirmar Pedido** and the form is valid, `Checkout.jsx` calculates the final totals before persisting the order:

```javascript
const subtotal = getCartTotal();

// Apply coupon discount
let discountAmount = 0;
if (appliedCoupon) {
  if (appliedCoupon.discountType === 'percentage') {
    discountAmount = subtotal * (appliedCoupon.discountValue / 100);
  } else {
    discountAmount = appliedCoupon.discountValue;
  }
  discountAmount = Math.min(discountAmount, subtotal);
}

const discountedSubtotal = subtotal - discountAmount;
const shipping = discountedSubtotal >= 100000 ? 0 : 15000; // Free shipping threshold
const tax = discountedSubtotal * 0.19;                     // IVA 19%
const total = discountedSubtotal + shipping + tax;
```

Shipping is free when the discounted subtotal reaches or exceeds **$100,000 COP**. Otherwise shipping is a flat **$15,000 COP**. Tax (IVA) is always **19%** of the discounted subtotal.

## Order data persistence

Before navigating to `/pago`, the full order payload is written to `localStorage`:

```javascript
const orderData = {
  orderNumber,      // "DD" + last 8 digits of Date.now()
  items,            // cartItems snapshot
  customerInfo,     // formData values
  subtotal,
  discount,         // discountAmount
  couponCode,       // appliedCoupon.code or null
  shipping,
  tax,
  total,
  date,             // new Date().toISOString()
};

localStorage.setItem('lastOrder', JSON.stringify(orderData));
```

The order number is generated as `"DD"` followed by the last 8 digits of the current Unix timestamp in milliseconds (e.g. `DD12345678`).

## After placing the order

After a successful payment step at `/pago`, the app navigates to `/confirmacion` (the `OrderConfirmation` page). On mount, that page:

1. Reads `lastOrder` from `localStorage`.
2. Calls `clearCart()` to empty the cart and remove the applied coupon.
3. Displays the order number, itemized receipt, shipping address, and a confirmation notice.
4. Removes `lastOrder` from `localStorage` on unmount.

If `lastOrder` is not found in storage when `/confirmacion` is loaded, the user is redirected to `/` automatically.
